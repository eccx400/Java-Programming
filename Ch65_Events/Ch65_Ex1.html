Assignment:

Write an application that shows a pair of dice. When the user clicks on the application window, the dice should be rolled (that is, the dice should be assigned newly computed random values). In addition, the application should also be able to respond to keyboard input. When the application has the input focus, it should be highlighted with a colored border, and the dice should be rolled whenever the user presses a key on the keyboard. This is in addition to rolling them when the user clicks the mouse on the application. Here is an applet that solves this exercise:

The application must be declared to "implement MouseListener", and it must define the mousePressed() method to roll the dice. (The other methods of the MouseListener interface will be empty.) The appropriate constructor must call addMouseListener(this) to register the panel to listen for mouse events on itself. For example,  the mousePressed() routine is defined simply as:

public void mousePressed(MouseEvent evt)
{
    roll();
}

The application also has to handle key presses, so it must implement the KeyListener interface, and it must register itself to listen for key events by calling addKeyListener() when it is constructed. The keyPressed() method just has to roll the dice, by calling the same roll() subroutine that is used by the mousePressed() routine.

The border around the dice changes from blue to red when the dice panel has the input focus. In order to implement this, the application must process focus events. So it needs to implement the FocusListener interface and register itself to listen for focus events. A boolean-valued instance variable, focussed, is set to true when the applet gains the focus and to false when the application loses the focus.

There is one thing that is easy to forget: On some platforms, an application does not automatically receive the input focus when the user clicks on it. It has to request the focus by calling the requestFocus() method. This is done in the mousePressed() routine, in response to a mouse click on the dice panel area. If you forget this, your application will work on some platforms, but on others it will never get the input focus and the keyboard will never work

A template application is provided which contains everything necessary for this application except the event handling code.
